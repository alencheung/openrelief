- **Quick Reference**: Immediate help during emergency situations
- **Language Accessibility**: Multi-language support for diverse communities

### Production Guidelines

#### Recording Standards

1. **Video Quality**
   - Resolution: 1920x1080 minimum
   - Frame rate: 30fps
   - Bitrate: 2500-4000 Kbps
   - Format: MP4 (H.264) for compatibility

2. **Audio Quality**
   - Sample rate: 48kHz
   - Bit depth: 16-bit
   - No background noise or echo
   - Consistent volume levels
   - Clear speech without clipping

3. **Content Standards**
   - Clear, slow-paced demonstrations
   - Step-by-step instructions
   - Multiple examples for complex tasks
   - Error prevention tips
   - Accessibility considerations

#### Recording Environment

1. **Lighting Setup**
   ```markdown
   ## LIGHTING GUIDELINES
   
   ### Three-Point Lighting
   - Key Light: Main light on subject/speaker
   - Fill Light: Soft light to reduce shadows
   - Back Light: Separates subject from background
   
   ### Softbox Lighting
   - Diffused light for even illumination
   - 45-degree angle from subject
   - Adjust distance for proper exposure
   - Use diffusion panel for softer light
   ```

2. **Audio Environment**
   ```markdown
   ## AUDIO RECORDING GUIDELINES
   
   ### Room Treatment
   - Reduce echo: blankets, curtains, carpets
   - Eliminate background noise: close windows, turn off fans
   - Consistent environment: same setup for all recordings
   
   ### Microphone Placement
   - 6-12 inches from speaker's mouth
   - Slightly off-axis to reduce plosives
   - Consistent distance throughout recording
   - Use pop filter for plosive reduction
   ```

### Tutorial Scripting

#### Script Structure
```markdown
## TUTORIAL SCRIPT TEMPLATE

### Introduction (30-60 seconds)
- Welcome and tutorial overview
- Learning objectives
- Prerequisites and setup requirements
- Estimated completion time

### Main Content (3-10 minutes)
- Step-by-step instructions
- On-screen demonstrations
- Common mistakes to avoid
- Tips and best practices

### Conclusion (30-60 seconds)
- Summary of key points
- Additional resources
- Next steps or related tutorials
- Call to action (practice, explore)
```

#### Script Writing Guidelines

1. **Clear and Concise Language**
   - Use simple, everyday language
   - Avoid technical jargon when possible
   - Define necessary technical terms
   - Use active voice ("Click the button" vs "The button should be clicked")

2. **Accessibility Considerations**
   - Describe visual elements for screen readers
   - Include keyboard alternatives
   - Mention accessibility features
   - Consider users with different abilities

3. **Emergency Context Awareness**
   - Consider stress levels during emergencies
   - Keep instructions calm and reassuring
   - Focus on critical information first
   - Provide multiple learning methods

## Accessibility in Tutorials

### Video Accessibility

#### Audio Descriptions
```markdown
## AUDIO DESCRIPTION GUIDELINES

### When to Include
- **Visual Content**: Describe important visual information
- **Text on Screen**: Read all text that appears
- **Actions and Transitions**: Describe movements and changes
- **Graphs and Charts**: Explain data visualizations

### Description Guidelines
- **Be Concise**: Describe what's essential for understanding
- **Be Objective**: Avoid personal opinions or interpretations
- **Timing**: Start descriptions slightly before visual appears
- **Pacing**: Match the pace of the visual content

### Audio Description Script
```
[Scene: Main dashboard appears]
